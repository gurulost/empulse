Required deploy steps:

1. Pull the latest code from `origin/main` so commit `a8e0866` is present.

2. Run:
```bash
php artisan migrate --force
```
Why:
- the onboarding reporting/telemetry features depend on the `onboarding_events` table from migration `2026_03_06_090000_create_onboarding_events_table.php`
- if this is missing, onboarding event writes and the Workfit admin onboarding report can fail

3. Run:
```bash
npm run build
```
Why:
- this release changes compiled frontend assets for:
  - the manager onboarding checklist and handoff states
  - the Workfit admin onboarding tab/banner
  - the employee dashboard trust block
  - the survey app trust block

4. Restart or redeploy:
- web app
- queue worker
- scheduler process
Why:
- workers can keep old PHP code loaded after deploy
- survey-wave and telemetry behavior needs the updated code everywhere, not just in the web process
